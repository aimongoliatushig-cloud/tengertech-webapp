Нэвтрэх

1. Нэвтрэх ба үндсэн зарчим

  • Нэвтрэх нэр: Odoo дээр бүртгэлтэй login эсвэл утас.
  • Нууц үг: тухайн хэрэглэгчийн Odoo password. Test үед admin password ашиглаж болно.
  • Нэвтэрсний дараа role-оос хамаарч sidebar/menu автоматаар өөрчлөгдөнө.
  • Browser notification зөвшөөрөл асуувал Allow дарна.
  • Block хийсэн бол Chrome Site settings → Notifications → Allow болгож refresh хийнэ.

3/15
Удирдлага

3. Захирал / Үйл ажиллагааны менежер

Харах

  • Бүх алба нэгжийн самбар
  • Хугацаа хэтэрсэн, хянах ажил
  • Тайлан статистик, жин

Хийх

  • Шинэ ажил үүсгэх
  • Ажил оноох
  • Хяналт дээр баталгаажуулах

Анхаарах

  • Role access зөв эсэх
  • Notification ирж байгаа эсэх
  • Бүх нэгжийн дата шинэ эсэх
4/15
Хэлтсийн дарга

4. Хэлтсийн дарга

  • Зөвхөн өөрийн хэлтсийн ажил, хүний нөөц, тайлан харагдана.
  • Шинэ ажил болон даалгавар үүсгэх эрхтэй.
  • Хэлтсийн даалгавруудын тайлан, явц, хяналтын төлөвийг харна.
  • Хүний нөөц хэсэг дээр зөвхөн өөрийн хэлтсийн хүмүүс харагдах ёстой.
  • Бусад хэлтсийн ажил/хүний мэдээлэл харагдвал access issue гэж тэмдэглэнэ.
5/15
Мастер

5. Ахлах мастер / мастер

  • Календарь болон ажлын жагсаалтаас өөрийн алба нэгжийн ажлуудыг харна.
  • Одоо байгаа ажлын дотор шинэ даалгавар нэмнэ.
  • Баг, хариуцах хүн, машин/маршрут шаардлагатай бол сонгоно.
  • Тайлан илгээж, шаардлагатай үед хяналт руу шилжүүлнэ.
  • Тохиргоо хэсэгт баг үүсгэж, багийн гишүүдийг сонгоно.
6/15
Ажилтан

6. Ажилтан / Mobile хэрэглэгч

  • Нэвтрээд зөвхөн өөрт оноогдсон ажил/даалгавар харагдана.
  • Мэдэгдэл дээр шинэ ажил ирсэн эсэхийг шалгана.
  • Даалгавар руу орж “Тайлан оруулах” дарна.
  • Тайлбар, хэмжээ/цэг, зураг, audio зэрэг шаардлагатай мэдээллээ хавсаргана.
  • Илгээсний дараа даалгавар хяналт хүлээж буй төлөвт шилжинэ.
7/15
Шинэ ажил

7. Шинэ ажил үүсгэх

1Sidebar эсвэл mobile dock дээр “Шинэ ажил” дарна.
2“Ажил нэмэх” сонгоно.
3Хэлтэс сонгоно. Хэлтсийн дарга/мастер дээр хэлтэс автоматаар сонгогдож болно.
4Ерөнхий ажил бол нэр, хариуцагч, төрөл, хугацаа оруулна.
5Хог тээвэрлэлт бол машин, маршрут, огноо сонгоно.
6“Ажил үүсгэх” дарж Odoo руу бүртгэнэ.
8/15
Даалгавар

8. Даалгавар үүсгэх

1Ажлын жагсаалт эсвэл календарь дээрээс үндсэн ажлаа нээнэ.
2“Даалгавар нэмэх” / “Өнөөдрийн ажил нэмэх” товч дарна.
3Даалгаврын нэр, огноо, хариуцсан хүн, баг сонгоно.
4Үүсгэсний дараа оноогдсон хүн дээр notification очих ёстой.
5Даалгавар detail руу орж мэдээлэл зөв хадгалагдсан эсэхийг шалгана.
9/15
Тайлан

9. Тайлан оруулах

1Өөрийн даалгавар дээр орно.
2“Тайлан оруулах” товч дарна.
3Гүйцэтгэлийн тайлбар, хувь, хэмжээ, цэг/тонн зэрэг мэдээллийг оруулна.
4Зураг болон audio хавсаргаж болно.
5Илгээх үед loading төлөв харагдана.
6Тайлан хяналт/баталгаажуулалтын урсгалд орно.
10/15
Хяналт

10. Хяналт ба баталгаажуулалт

  • Ажилтан тайлан илгээсний дараа мастер/хэлтсийн дарга/менежерийн хяналт руу орно.
  • Хянах ажил дээр орж тайлан, зураг, явцыг шалгана.
  • Зөв бол “Дууссан” эсвэл баталгаажуулах action хийнэ.
  • Дутуу бол тайлбар үлдээж буцаах/засуулах урсгал ашиглана.
  • Dashboard дээр хянагдаж байгаа, хугацаа хэтэрсэн, дууссан ажлын тоо шинэчлэгдэнэ.
11/15
Хог тээвэрлэлт

11. Хог тээвэрлэлтийн workflow

1Авто бааз, хог тээвэрлэлтийн хэлтэс сонгоно.
2Хог тээвэрлэлтийн маршрут горим сонгоно.
3Машин сонгоно.
4Маршрут/хогийн цэгийн багц сонгоно.
5Баг сонгож, багийн гишүүд болон хариуцсан хүмүүсээ шалгана.
6Үүсгэхэд үндсэн ажил болон маршрутын цэг бүрийн даалгавар автоматаар үүснэ.
12/15
Статистик

12. Тайлан, статистик ашиглах

  • Тайлан, статистик хэсэгт хэлтэс болон доторх нэгжээр шүүнэ.
  • Хог тээвэрлэлт дээр машины өдөр тутмын жин, нийт кг, хугацааны багц харагдана.
  • Өнөөдрийн явц дээр явж буй, хяналт хүлээж буй, дууссан ажил тусдаа харагдана.
  • Захирал/үйл ажиллагааны менежер бүх нэгжийг харна.
  • Хэлтсийн дарга/мастер өөрийн хүрээний тайланг л харна.
13/15
Календарь

13. Календарь ба мэдэгдэл

  • Календарь дээр долоо хоног болон сарын view-ээр ажил харагдана.
  • Timeline/card дээр дарахад тухайн даалгаврын дэлгэрэнгүй рүү орно.
  • Bell icon дээр дарж өөрт ирсэн мэдэгдлүүдийг харна.
  • Шинэ ажил/даалгавар оноогдоход notification ирэх ёстой.
  • Browser notification зөвшөөрөл аваагүй бол Chrome permission popup дээр Allow дарна.
14/15
Checklist

14. Өдөр тутмын шалгах checklist

  • Зөв хэрэглэгчээр нэвтэрсэн эсэх
  • Зөвхөн зөв хэлтэс/өөрийн ажил харагдаж байгаа эсэх
  • Шинэ ажил/даалгавар үүсгэх товч зөв role дээр харагдаж байгаа эсэх
  • Оноогдсон хүнд notification очиж байгаа эсэх
  • Тайлан илгээхэд loading, success, төлөв шинэчлэлт зөв эсэх
  • Хяналт дээр дууссан төлөв зөв хадгалагдаж байгаа эсэх
  • Хог тээвэрлэлтийн машин/маршрут/жин data зөв татагдаж байгаа эсэх
