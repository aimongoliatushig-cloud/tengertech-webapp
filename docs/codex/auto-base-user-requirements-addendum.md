# Авто бааз — нэмэлт хэрэглэгчийн шаардлага

## 1. Машин техникийн төрөл

Авто бааз дээр бүртгэгдэх машин техник нь төрөлтэй байна.

Жишээ төрөл:

- Хогны машин
- Ковш
- Өргөгч
- Усалгааны машин
- Ачааны машин
- Тусгай зориулалтын техник
- Бусад

Систем дээр машин техникийн төрлийг тохиргооноос нэмэх, засах, идэвхгүй болгох боломжтой байна.

## 2. Машин техникийн бүртгэл

Машин техник бүр дээр дараах мэдээлэл бүртгэгдэнэ.

| Талбар | Тайлбар |
|---|---|
| Улсын дугаар | Машины улсын дугаар |
| Машин техникийн нэр | Жишээ: Хог тээврийн машин 01 |
| Төрөл | Хогны машин, ковш, өргөгч гэх мэт |
| Марк / модель | Машины техникийн мэдээлэл |
| Үйлдвэрлэсэн он | Машины үйлдвэрлэсэн он |
| Хэлтэс | Тухайн машины харьяалагдах хэлтэс |
| Хариуцсан жолооч | Хэлтсийн дарга онооно |
| Төлөв | Ажиллаж байгаа, засвартай, эвдэрсэн, ашиглалтаас гарсан |
| Даатгалын мэдээлэл | Даатгалын компани, гэрээний дугаар, эхлэх/дуусах огноо |
| Улсын үзлэгийн мэдээлэл | Сүүлд орсон огноо, дараагийн орох огноо |
| Засварын түүх | Тухайн машин дээр хийгдсэн бүх засвар |

## 3. Жолооч оноох

Машин техник бүрт хариуцсан жолооч оноогдоно.

Энэ үйлдлийг зөвхөн:

- Хэлтсийн дарга
- Системийн админ

хийх боломжтой байна.

Хэлтсийн дарга тухайн машин техникийн хариуцсан жолоочийг сонгож онооно.

Жолооч солигдсон тохиолдолд өмнөх жолоочийн түүх хадгалагдана.

Жолоочийн хариуцалтын түүх дээр:

- Хэн жолооч байсан
- Хэзээнээс хэзээ хүртэл хариуцсан
- Хэн өөрчилсөн
- Өөрчилсөн огноо

харагдана.

## 4. Даатгалын мэдээлэл

Машин техник бүр дээр даатгалын мэдээлэл бүртгэгдэнэ.

Даатгалын талбарууд:

| Талбар | Тайлбар |
|---|---|
| Даатгалын компани | Даатгал хийсэн байгууллага |
| Даатгалын гэрээний дугаар | Полис / гэрээний дугаар |
| Даатгал эхлэх огноо | Даатгал хүчинтэй болох огноо |
| Даатгал дуусах огноо | Даатгал дуусах огноо |
| Даатгалын баримт | PDF / зураг хавсаргах боломжтой |
| Тайлбар | Нэмэлт тэмдэглэл |

## 5. Улсын үзлэгийн мэдээлэл

Машин техник бүр дээр улсын үзлэгийн мэдээлэл орно.

Талбарууд:

| Талбар | Тайлбар |
|---|---|
| Улсын үзлэгт орсон огноо | Сүүлд үзлэгт орсон өдөр |
| Дараагийн үзлэгт орох огноо | Дараа жилийн үзлэгийн хугацаа |
| Үзлэгийн баримт | PDF / зураг хавсаргах боломжтой |
| Тайлбар | Нэмэлт тэмдэглэл |

## 6. Даатгал болон улсын үзлэгийн сануулга

Системийн тохиргоонд сануулах өдрийн тоо оруулдаг байна.

Жишээ:

```text
Даатгал дуусахаас 30 хоногийн өмнө сануулах
Улсын үзлэг болохоос 14 хоногийн өмнө сануулах
```

Сануулах өдрийн тохиргоог админ эсвэл эрхтэй хэрэглэгч өөрчилж болно.

Сануулах өдөр болоход хэлтсийн даргад мэдэгдэл очно.

Мэдэгдэлд дараах мэдээлэл харагдана:

- Машины улсын дугаар
- Машины төрөл
- Хариуцсан жолооч
- Ямар хугацаа дуусах гэж байгаа
- Даатгал
- Улсын үзлэг
- Дуусах огноо
- Үлдсэн хоног
- Машины дэлгэрэнгүй рүү орох товч

Жишээ мэдэгдэл:

```text
12-34 УБА улсын дугаартай хогны машины даатгал 2026-06-01 өдөр дуусна. Үлдсэн хоног: 30.
```

## 7. Засварын түүх

Машин техник бүр дээр засварын түүх харагдана.

Засварын түүх дээр:

| Талбар | Тайлбар |
|---|---|
| Засварын огноо | Засвар эхэлсэн / дууссан огноо |
| Эвдрэлийн төрөл | Ямар эвдрэл гарсан |
| Засварын тайлбар | Юу зассан |
| Сэлбэг ашигласан эсэх | Ашигласан сэлбэгийн мэдээлэл |
| Зардал | Засварын нийт дүн |
| Хариуцсан механик | Засвар хийсэн хүн |
| Засварын төлөв | Үүссэн, хийгдэж байгаа, дууссан |
| Баримт / зураг | Засварын өмнө ба дараах зураг, PDF |
| Худалдан авалттай холбоотой эсэх | Procurement workflow-той холбогдсон эсэх |

Машины дэлгэрэнгүй хуудас дээр `Засварын түүх` гэсэн tab байна.

## 8. Авто баазын худалдан авалт

Авто баазын худалдан авалт шинээр тусдаа workflow хийхгүй.

Өмнөх pull request-ээр орж ирсэн existing procurement workflow-г ашиглана.

Авто баазын засварын үед сэлбэг, үйлчилгээ, материал худалдан авах шаардлагатай бол тухайн засварын ажлаас procurement request үүсгэнэ.

Workflow:

```text
Засварын хүсэлт
    ↓
Сэлбэг / үйлчилгээ хэрэгтэй гэж тэмдэглэх
    ↓
Existing procurement workflow ашиглаж худалдан авалт үүсгэнэ
    ↓
Нярав / худалдан авалт 3 үнийн санал оруулна
    ↓
Санхүү төлбөр бүртгэнэ
    ↓
Сэлбэг хүлээн авна
    ↓
Механик засвар хийж дуусгана
    ↓
Засварын түүх машин дээр хадгалагдана
```

Procurement record нь тухайн машин техник болон засварын хүсэлттэй холбоотой байна.

## 9. Хог ачалтын машины өдөр тутмын жингийн тайлан

Хог ачалтын машин бүрийн жингийн тайланг өдөр бүр тохируулсан цагт автоматаар татна.

Энэ нь зөвхөн `Хогны машин` төрөлтэй машин дээр ажиллана.

Татах мэдээлэл:

| Талбар | Тайлбар |
|---|---|
| Огноо | Тайлангийн огноо |
| Машины улсын дугаар | Аль машин |
| Жин | Хог ачсан жин |
| Нэгж | кг / тонн |
| Эх сурвалж | Гадны систем |
| Татсан огноо | Систем татсан цаг |
| Төлөв | Амжилттай / алдаатай |
| Алдааны мэдээлэл | Хэрэв таталт амжилтгүй бол |

Жингийн тайлан нь машин техникийн дэлгэрэнгүй дээр харагдана.

Мөн хог тээвэрлэлтийн тайлан, dashboard дээр ашиглагдана.

## 10. Хог ачалтын машины өдөр тутмын шатахууны мэдээлэл

Хог ачалтын машин бүрийн ашигласан шатахууны мэдээллийг өдөр бүр автоматаар татна.

Энэ нь зөвхөн `Хогны машин` төрөлтэй машин дээр ажиллана.

Татах мэдээлэл:

| Талбар | Тайлбар |
|---|---|
| Огноо | Тайлангийн огноо |
| Машины улсын дугаар | Аль машин |
| Зарцуулсан шатахуун | Литрээр |
| Түлшний төрөл | Дизель / бензин гэх мэт |
| Эх сурвалж | Гадны систем |
| Татсан огноо | Систем татсан цаг |
| Төлөв | Амжилттай / алдаатай |
| Алдааны мэдээлэл | Хэрэв таталт амжилтгүй бол |

Шатахууны мэдээлэл нь машин техникийн дэлгэрэнгүй дээр харагдана.

Мөн авто баазын зардлын тайлан, машин ашиглалтын тайлан дээр ашиглагдана.

## 11. Гадны системээс мэдээлэл татах тохиргоо

Хог ачалтын жин болон шатахууны мэдээллийг татах web хаяг, username, password нь `.env` файлд хадгалагдсан байна.

Код дотор username, password шууд бичихгүй.

`.env` жишээ:

```env
GARBAGE_WEIGHT_API_URL=
GARBAGE_FUEL_API_URL=
GARBAGE_API_USERNAME=
GARBAGE_API_PASSWORD=
```

Систем `.env`-ээс мэдээллийг уншиж гадны системтэй холбогдоно.

Алдаа гарсан тохиолдолд:

- Алдааны log хадгална
- Сүүлийн амжилттай татсан огноог хадгална
- Хэлтсийн даргад таталт амжилтгүй болсон мэдэгдэл өгнө
- Дараагийн scheduled run дээр дахин оролдоно

## 12. Автомат таталтын тохиргоо

Систем дээр автомат таталтын цаг тохируулдаг байна.

Жишээ:

```text
Өдөр бүр 20:00 цагт хог ачалтын жин татах
Өдөр бүр 20:30 цагт шатахууны мэдээлэл татах
```

Тохиргоонд:

| Тохиргоо | Тайлбар |
|---|---|
| Жингийн мэдээлэл татах эсэх | Идэвхтэй / идэвхгүй |
| Жингийн мэдээлэл татах цаг | Өдөр бүр хэдэн цагт |
| Шатахууны мэдээлэл татах эсэх | Идэвхтэй / идэвхгүй |
| Шатахууны мэдээлэл татах цаг | Өдөр бүр хэдэн цагт |
| Сануулгын хоног | Даатгал / үзлэгийн сануулга хэдэн хоногийн өмнө очих |

## 13. Хэлтсийн даргад харагдах dashboard

| Нийт машин техник | Бүх бүртгэлтэй машин |

| Засвартай машин | Засварт байгаа |

| Улсын үзлэг болох гэж байгаа | Сануулах хугацаанд орсон машинууд |

## 15. Эрхийн шаардлага

### Хэлтсийн дарга

Хэлтсийн дарга дараах эрхтэй байна.

- Машин техник бүртгэх
- Машин техникийн мэдээлэл засах
- Машины төрөл сонгох
- Жолооч оноох
- Даатгалын мэдээлэл оруулах
- Улсын үзлэгийн мэдээлэл оруулах
- Засварын хүсэлт үүсгэх / хянах
- Procurement workflow-той холбох
- Засварын түүх харах
- Жингийн тайлан харах
- Шатахууны тайлан харах
- Сануулга хүлээн авах

### Жолооч

Жолооч дараах мэдээллийг харна.

- Өөрт оноогдсон машин
- Машины үндсэн мэдээлэл
- Засварын хүсэлт илгээх боломж
- Өөрийн машинтай холбоотой засварын төлөв
- Өдөр тутмын ажил / маршрут

### Механик

Механик дараах эрхтэй байна.

- Засварын хүсэлт харах
- Засварын ажил гүйцэтгэх
- Засварын тайлан оруулах
- Засварын зураг, баримт хавсаргах
- Ашигласан сэлбэг тэмдэглэх
- Засвар дуусгах

### Нярав / худалдан авалт

Нярав нь existing procurement workflow дээр ажиллана.

- Сэлбэгийн худалдан авалт хянах
- Үнийн санал оруулах
- Сэлбэг хүлээн авах
- Procurement record-ийг засварын ажилтай холбох

### Санхүү

Санхүү нь existing procurement workflow дээр ажиллана.

- Төлбөр төлсөн мэдээлэл бүртгэх
- Төлбөрийн баримт хавсаргах
- Худалдан авалтын төлөв шинэчлэх

## 16. Гол бизнес дүрэм

1. Машин техник бүр төрөлтэй байна.
2. Машин техник бүрт хариуцсан жолооч оноогдоно.
3. Жолоочийг хэлтсийн дарга онооно.
4. Машин техник бүр даатгалын мэдээлэлтэй байж болно.
5. Машин техник бүр улсын үзлэгийн мэдээлэлтэй байж болно.
6. Даатгал болон улсын үзлэгийн хугацаа болохоос өмнө систем сануулга өгнө.
7. Сануулах өдрийн тоо тохиргооноос удирдагдана.
8. Машин техник бүрийн засварын түүх хадгалагдана.
9. Авто баазын худалдан авалт нь existing procurement workflow-г ашиглана.
10. Хогны машин бүрийн хог ачалтын жингийн тайлан өдөр бүр автоматаар татагдана.
11. Хогны машин бүрийн шатахууны мэдээлэл өдөр бүр автоматаар татагдана.
12. Гадны системийн URL, username, password нь `.env` файлд хадгалагдана.
13. Код дотор нууц мэдээлэл hardcode хийхгүй.
14. Таталт амжилтгүй бол log үүсэж, хэлтсийн даргад мэдэгдэнэ.

Товчоор хэлбэл одоо авто баазын шаардлага илүү бүрэн болж байна:

**машин техникийн төрөл + жолооч оноолт + даатгал + улсын үзлэг + сануулга + засварын түүх + procurement workflow холболт + хогны машины өдөр тутмын жин таталт + шатахуун таталт** гэсэн үндсэн хэсгүүдтэй боллоо.
